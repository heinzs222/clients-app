1. Introduction

Welcome to CAPI Launcher. This Privacy Policy outlines our practices regarding the collection, use, and disclosure of information when you use our service to provision and manage serverless endpoints for the Meta Conversions API. We are committed to protecting your privacy and ensuring transparency in how we handle data, particularly sensitive credentials like Meta Dataset IDs and Access Tokens.

database

2. Data Collection

When you use CAPI Launcher, we collect specific configuration data necessary to deploy your serverless endpoints. This includes:

  • Meta Dataset IDs (Pixel IDs): Used to route events to the correct Meta dataset.
  • Conversion API Access Tokens: Required to authenticate server-to-server requests with Meta.
  • Account Information: Basic profile details used for authentication and account management.
  • Endpoint Configuration: Domain names, routing paths, and specific event configurations you set up via our UI.
route

3. Data Usage & Routing

The core function of CAPI Launcher is to act as a secure, high-performance conduit for your marketing data. We use the collected information exclusively to:

  • Provision serverless infrastructure on your behalf.
  • Route incoming HTTP requests (events) from your web assets directly to Meta's Graph API.
  • Provide operational metrics and debugging logs within the CAPI Launcher dashboard.

Important Note: CAPI Launcher does not persistently store the payload of the events (e.g., user details, purchase amounts) passing through your endpoints. We act purely as a pass-through layer to ensure compliance and minimize data residency risks.
